# Plan: L1 Fase 6 — UI da seleção nacional

**Epic:** l1-national · **Spec:** [l1](../specs/2026-06-20-l1-national-team-design.md) §6 · **Data:** 2026-06-26
**Pré-requisito:** L1 backend Fases 1–5 completo (`676016c`). Esta é a última fase do L1.

**Ressalva de QA (importante):** Playwright MCP indisponível nesta sessão → as telas são construídas no kit do
Design System e testadas por **snapshot + lógica** no UI-jest, mas a **validação visual no browser é handoff manual**.
Type-check e snapshot não garantem UX (regra do projeto). Seguir o kit + telas já validadas reduz o risco.

## Invariantes de UI (D0–D8)
- Componentes do **kit** (`src/components/kit/`: Card/Button/Chip/Badge/Icon/EmptyState/Sheet/Skeleton/Toast/useConfirm)
  + tipografia semântica (`src/components/typography`). **Cores/spacing via tokens** (`src/theme`), nunca hex hardcoded.
  Accent do clube via ClubAccentProvider quando fizer sentido.
- **`useConfirm`, nunca `Alert.alert`** (no-op no RN Web). i18n pt/en com **paridade** (`national.*`).
- Telas: PascalCase, 1 componente/arquivo em `src/screens/national/`. Acessibilidade nos elementos interativos.
- Snapshot tests no UI-jest (`jest.ui.config.js`) — padrão das telas migradas em D5.

## Dados disponíveis (backend pronto — só consumir)
`getUserManagedNation`/`loadNationalTeams`/`setUserManagedNation`; `loadNationalFixtures`/`loadNationalFixturesByCompetition`
+ `calculateStandings` (engine) p/ tabela; `getCallUps`/`setManualCallUp`/`clearWindowCallUps` + `deriveNationalPool`
(+ query de jogadores por nacionalidade do L1-B) p/ convocação; `getNationalTitles`; `getCaps` (+ query de líderes se preciso).

## Telas
1. **Hub — migrar `InternationalsScreen` ao kit.** Vira o hub da seleção: nação dirigida + força + pontos de entrada
   (Convocação/XI, Calendário/Tabela, Histórico). Hoje mistura `@/theme` cru com kit — completar a migração (remover
   `StyleSheet`/`commonStyles` hardcoded; usar tokens). Manter a listagem de elegíveis do elenco do usuário como contexto.
2. **`NationalSquadScreen` (Convocação + XI).** Pool da seleção dirigida; convocação auto marcada; **override manual**
   interativo (Chip/Sheet → `setManualCallUp`/`clearWindowCallUps`) com `useConfirm`; preview do XI inicial (top-11).
3. **`NationalCalendarScreen` (Calendário + Tabela).** Fixtures da eliminatória/torneio (`loadNationalFixtures`) +
   classificação (`calculateStandings`) + chaveamento do mata-mata. Empty state fora de janela/ciclo.
4. **`NationalHistoryScreen` (Histórico).** Títulos (`getNationalTitles`) + líderes de caps/gols (`getCaps`; adicionar
   `getTopCaps` com `__rowSchemas` se necessário). Empty state quando ainda sem títulos.

## Navegação
Registrar as 3 telas novas em `src/navigation/RootNavigator.tsx` + `types.ts`; linkar a partir do hub. `nav.*` keys i18n.

## Não-objetivos
Sem mudança de engine/balanceamento/schema (só leitura; exceção: possível `getTopCaps` aditivo). Sem novo épico.

## Execução
1 workflow: implementer(high, telas+nav+i18n+snapshot tests) → verify(tsc + node + UI jest + parity) → review (kit/tokens,
useConfirm, i18n parity, nav registrada, a11y, sem Alert.alert) → fix. **Handoff:** listar as telas p/ o usuário validar no browser.

## Gate
`tsc` 0 · `npm test` (node+ui) verde · snapshots novos · parity verde · `Alert.alert` em src/screens/national = 0.
