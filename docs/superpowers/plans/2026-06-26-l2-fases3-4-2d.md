# Plan: L2 Fases 3–4 — superfície visual 2D (mapas + opt-in)

**Epic:** l2-match-2d · **Spec:** [l2](../specs/2026-06-20-l2-match-engine-2d-design.md) §6 · **Data:** 2026-06-26
**Pré-requisito:** L2 Fases 1–2 (`0dd38c9`) — geometria `(x,y)`/`phase`/`xg` persistida em `match_events` e legível
via `getMatchEvents` (retorna x/y/phase/xg). **Sem mudança de engine** nesta entrega — puro consumo de dados.

**Ressalva de QA (firme):** Playwright indisponível → componentes SVG construídos no padrão `RadarChart` + tokens +
**snapshot tests** (estrutura guardável), mas o **polimento/legibilidade visual é handoff manual no browser**. A geometria
é determinística e testada (coords em [0,1], gols no terço certo), então a correção estrutural é alcançável; só a estética precisa de olho.

## Escopo
- **Fase 3 — mapas estáticos (L2.5):** `Pitch2D` (campo SVG), `ShotMap` (chutes por (x,y); raio ∝ xG; cor por resultado:
  gol/no-alvo/fora/defendido), `HeatMap` (densidade de posições por grid). Em `src/components/`, react-native-svg + tokens
  `@/theme` + kit. Helpers PUROS de view-model (extrair chutes dos eventos, binning do heatmap) — unit-testados.
- **Fase 4 — integração opt-in (L2.7):** seção/toggle "Ver em 2D" em `MatchResultScreen` (JÁ no kit). Default OFF (casual
  vê só o resumo). Quando ON: carrega `getMatchEvents(fixtureId)` da partida do usuário e renderiza os mapas. Preferência
  persistida (ui-store/settings-store). i18n pt/en paridade.

## Fora de escopo (deferido)
- **PassNetwork:** aproximada até L2.2 (precisa de pares passador→recebedor; só `assist` tem secondaryPlayerId). Fora.
- **Fase 5 — replay temporal:** animação rAF/timeline — mais dependente de browser (timing). Fora desta passada.
- Qualquer mudança de engine/balanceamento/determinismo. Geometria de AI (só a do usuário tem geometria).

## Riscos
- **Visual sem QA:** mitigado por (a) dados determinísticos testados, (b) projeção mecânica [0,1]→retângulo do campo,
  (c) snapshot tests, (d) handoff explícito. Risco residual = estética/legibilidade, não correção estrutural.
- **Fonte dos eventos:** o resultado in-memory tem `xg` mas NÃO x/y/phase (derivados só na persistência). A tela DEVE ler
  via `getMatchEvents(fixtureId)` (DB) p/ ter a geometria — não confiar no objeto in-memory do store.
- **Toggle reversível:** com OFF, a tela deve ficar idêntica à atual (casual intocado).

## Execução
1 workflow TDD: implementer(high, componentes+helpers+integração+snapshot/unit tests)→verify(tsc+node+ui+parity)→
review(SVG no kit/tokens, opt-in reversível, lê do DB, pureza dos helpers, a11y, i18n)→fix. **Handoff:** listar telas/mapas
p/ validação visual no browser.

## Gate
`tsc` 0 · `npm test` (node+ui) verde · snapshots dos mapas · parity verde · toggle OFF = tela idêntica · sem mudança de engine.
